今天完成 LeetCode 33题. (其实还应该有31题,然而,真的没看懂题目) 以下是我的解法(PHP版本) , 还有因为各人喜好问题,并没有用递归.

function search($nums,$target)
{
    // 还有一个直接foreach查找的,虽然时间复杂度是 O(n),但实际执行比这个快!!! 

    // LogN的复杂度,需要用到判断条件有点复杂的折半查找

    //每次折成两半之后,都会有一个是有序数组,另一个是无序的,那么有序的好判断,没有在有序数组中,就继续折半查找无序的部分
    // 直接上4指针?还是递归?
    $low = 0;
    $heigh = count($nums)-1;
    while ($low <= $heigh) {
        // 情况1
        if ($nums[$low]>$target && $nums[$heigh] < $target) {
            // 不用继续找了,肯定没有..
            return -1;
        }
        // 分两半 
        $mid = round(($heigh-$low)/2)+$low;
        echo "{$low}:{$mid}:{$heigh}\n";
        // 看看运气,是否刚好找到了?
        if ($nums[$mid] == $target) {
            return $mid;
        }
        if ($nums[$low] == $target) {
            return $low;
        }
        if ($nums[$heigh] == $target) {
            return $heigh;
        }
        // 特殊的,判断到这里就要看看是否两个指针已经交汇了
        if($low == $heigh || $mid == $low || $mid == $heigh){
            return -1;
        }
        // 当然通常运气不好那么好直接找到,那么判断左右两边,哪边刚好是正常的数组
        if($nums[$low] <= $nums[$mid] ){
            // 左边是正常数组
            // 看看目标是否在正常数组中
            if($nums[$low]< $target && $target < $nums[$mid]){
                // 没的说,继续找的就是这个数组了.
                $heigh = $mid;// 高指针如果刚好等于中指针这种情况
            }else{
                // 不幸的在另一边或没有..
                $low = $mid;
            }
        }else{
            // 右边是正常数组
            if ($nums[$mid]<$target && $target < $nums[$heigh]) {
                // 在正常数组中
                $low = $mid;
            }else{
                $heigh = $mid;
            }
        }
    }
    
    return -1;
}

    • sweet angels 可爱的小天使

    • 熟睡的,酣睡的:
      • sound asleep
      • fast asleep
      • sleeping deeply
    • I’m surprised to see you awake. Two miinutes ago ,you were fast/sound asleep. 真奇怪你怎么醒了?两分钟前你还睡得很香呢.

    • sb’s heart melt / melt sb’s heart 使某人心软,态度软化,心被融化

  • You don’t resent the rich, that’s not your sensibility. 你并不仇富,仇富不是你的态度.

    • resent (doing) sth. 反感/憎恶 (做)某事
